Application Python Filtre UART

Objectif de l'application : Les produits que je suis amené à tester dans mon entreprise communiquent leurs actions par UART sur mon PC. Cependant lors de tests spécifiques, seul certaines informations sont intérressantes et sont utilises à savoir en temps réel. Pour cela j'ai dévelloppé ce programme qui lit sur le port USB les informations donnés par le produit et les filtre avec les paramètres renseignés.

1. Architecture Globale

J'utilise la bibliothèque tk pour créer une interface graphique avec laquel l'utilisateur va pouvoir visualiser les traces en direct et renseigner ses paramètres. Pour fluidifier le fonctionnemt de l'application, je déclenche les fontions par thread. Cela me permet de donner un fonctionnemt asynchrone à mon application et ne pas la faire freezer.

2. Découpage et Analyse des Composants

La méthode __init__ configure la fenêtre et initialise toutes les variables d'état. Elle appelle ensuite les méthodes de création de l'interface.

Gestion Asynchrone : L'utilisation de threading.Event() pour self.read_thread_event permet de gérer l'arrêt sécurisé du thread de lecture.

  • Barre d'état : Affiche les paramètres en cours d'utilisation (port de lecture, vitesse, activation du filtre). L'utilisation d'un Canvas pour avoir un retour visuel sur l'étata de lecture(led rouge ou verte).
  • Zone de texte : Utilise ScrolledText pour afficher les traces voulu. ScrolledText est nécéssaire pour les grandes quantités de texte et permet de naviguer facilement à travers.
  • Onglets : Les onglets dépliables ouvrent des fenêtres permettant à l'utilisateur de sélectionner ses paramètres. J'utilise pour cela la classe Menu de tk

Gérer des lectures série est délicat car la fonction ser.readline() est bloquante.
J'effectue donc la lecture en deux parties :

  • Threading : Déporter la lecture dans un processus séparé (threading.Thread) empêche l'interface de figer.
  • Traitement : Les données sont ensuite décodées et si une erreur intervient dans le décodage, errors='replace' permet d'écrire l'erreur à la place des données. Les donnée sont ensuites filtrés puis affichés et écrites dans un fichier de log.

Filtrage : L'option de filtrage consiste à vérifier dans chaque ligne si un mot clée est présent. Si c'est le cas alors la ligne est affichée. L'utilisateur peut renseigner plusieurs mots clée et choisir entre l'option ET pour vérifier tous les mots soient présent sur la ligne ou OU pour vérifier qu'un des mots soit présent sur la ligne.

Paramètres : Dans la fenêtre paramètre, il est possible de sélectionner le port à lire ainsi que la vitesse de lecture. Pour ne perdre aucune information, toutes les traces lu sur le port sont écrites dans un fichier de log. Il est possible dans le menu déroulant paramètre de sélectionner le nom du fichier ainsi que son emplacement.

Le filtre n'étant pas actif, toutes les informations lu sur le port 5 seront affichés.
