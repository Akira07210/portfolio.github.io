Application Python Filtre UART

Objectif de l'application : Les produits que je teste communiquent leurs actions via une liaison UART vers mon PC. Cependant, lors de certains tests spécifiques, seules certaines informations sont pertinentes et doivent être exploitées en temps réel. J’ai donc développé ce programme qui lit les données via le port USB, puis les filtre en fonction des paramètres définis par l’utilisateur.

1. Architecture globale

J’utilise la bibliothèque tk pour créer une interface graphique permettant à l’utilisateur de visualiser les traces en temps réel et de configurer les paramètres. Pour améliorer la fluidité de l’application, j’utilise des threads afin d’exécuter certaines fonctions en parallèle. Cela permet d’avoir un fonctionnement asynchrone et d’éviter le blocage de l’interface.

2. Découpage et analyse des composants

La méthode __init__ configure la fenêtre et initialise toutes les variables d’état. Elle appelle ensuite les méthodes de création de l’interface.

Gestion asynchrone : L’utilisation de threading.Event() pour self.read_thread_event permet de gérer l’arrêt sécurisé du thread de lecture.

  • Barre d’état : Affiche les paramètres en cours d’utilisation (port, vitesse, activation du filtre). Un Canvas est utilisé pour fournir un retour visuel de l’état de lecture (LED rouge ou verte).
  • Zone de texte : Utilise ScrolledText pour afficher les traces souhaitées. Cet outil est adapté aux grandes quantités de données et permet une navigation fluide.
  • Onglets : Les menus déroulants permettent à l’utilisateur de configurer ses paramètres. J’utilise pour cela la classe Menu de tk.

La lecture sur un port série est délicate car la fonction ser.readline() est bloquante. J’ai donc séparé le traitement en deux parties :

  • Threading : La lecture est effectuée dans un thread séparé (threading.Thread), ce qui empêche le blocage de l’interface.
  • Traitement : Les données sont décodées, et en cas d’erreur, l’option errors='replace' permet de remplacer les caractères invalides. Les données sont ensuite filtrées, affichées et enregistrées dans un fichier de log.

Filtrage : Le filtrage consiste à vérifier si un mot-clé est présent dans chaque ligne. Si c’est le cas, la ligne est affichée. L’utilisateur peut définir plusieurs mots-clés et choisir entre : ET (tous les mots doivent être présents) ou OU (au moins un mot doit être présent).

Paramètres : L’utilisateur peut sélectionner le port série et la vitesse de lecture. Toutes les données lues sont enregistrées dans un fichier log afin de ne perdre aucune information. Il est également possible de définir le nom du fichier et son emplacement.

3. Aperçu en fonctionnement

Lecture sans filtre

Le filtre n’étant pas activé, toutes les données lues sur le port 5 sont affichées.

Lecture avec filtre

Le mot recherché est M5. Le filtre permet donc d’afficher uniquement les trames correspondant à ce critère.
