Automated Request Luna

Objectif de l'application : Certains des appareils que j’ai testés sont supervisés via une plateforme web. Mon objectif a donc été d’automatiser les actions sur ces appareils via cette plateforme, puis de traiter les résultats remontés.

⚠️

Pour des raisons de confidentialité, le code source et le fonctionnement exact de ce programme ne sont pas partagés.
Merci de votre compréhension.

1. Fonctionnement général

Mon application (nommée ARL) s’authentifie sur le serveur afin de superviser les appareils souhaités. Une fois que les produits en test ont terminé les tâches demandées, l’ARL se reconnecte à la plateforme pour récupérer les résultats. Les données sont ensuite triées puis stockées de manière claire dans un fichier log afin que le testeur puisse identifier rapidement les informations importantes.

2. Découpage et analyse des composants

Grâce à l’interface graphique, l’utilisateur peut facilement renseigner les identifiants des produits à tester. Ces identifiants sont enregistrés et réutilisés dans les requêtes envoyées au serveur.

Pour communiquer avec la plateforme web, j’utilise les bibliothèques BeautifulSoup pour analyser les réponses HTML et requests pour envoyer les requêtes au serveur.

Pour chaque requête, l’ARL vérifie le code de réponse HTTP, car le serveur n’est pas toujours joignable. Si une requête ne retourne pas un code 200 ou 201, l’erreur est enregistrée dans les logs et une nouvelle tentative est effectuée après un délai.

Après un délai défini par l’utilisateur (afin de laisser le temps au produit d’exécuter la tâche), l’ARL se reconnecte à la plateforme pour récupérer les résultats. Seules les informations pertinentes pour le test sont conservées.

Les informations principales (comme le résultat de la requête) sont affichées directement dans l’interface graphique, permettant une lecture rapide de l’état du test. L’ensemble des données est également enregistré dans un fichier log.csv.

En cas de doute, il est possible de vérifier manuellement les résultats grâce à l’identifiant de la requête enregistré dans les logs (gestion des erreurs).

Afin de garantir un fonctionnement continu du programme, l’utilisation des blocs try { } except { } est fréquente. Ils permettent de gérer les erreurs sans interrompre l’exécution du programme. Les erreurs sont également affichées dans l’interface graphique.
